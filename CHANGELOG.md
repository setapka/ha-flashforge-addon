# Changelog

Все изменения в проекте Flashforge Adventurer 5M Control Panel.

## [1.3.0] - 2026-06-29

### Исправления проблем с автообнаружением и подключением

#### Добавлено
- **Конфигурация подсети**: Новая опция `scan_subnet` в конфигурации аддона для указания подсети по умолчанию
- **Автоподключение принтеров**: При указании IP принтера в конфигурации, подключение происходит автоматически при старте
- **Подробное логирование**: Детальное логирование всех операций для упрощения отладки

#### Изменения
- **main.py**:
  - `DiscoveryService.scan_subnet`: Улучшена обработка ошибок, добавлено логирование количества проверенных хостов и найденных принтеров
  - `handle_discovery`: Поддержка параметра `subnet` из query string, значение по умолчанию из переменной окружения `SCAN_SUBNET`
  - `FlashforgeClient.connect`: Раздельная обработка TimeoutError, ClientError и других исключений
  - `FlashforgeClient._parse_status`: Логирование изменений состояния принтера
  - `handle_add_printer`: Поддержка обоих имён параметра (printer_port и moonraker_port)
  - `_auto_connect_config_printers`: Новый метод для автоподключения к принтеру из конфига
  - `run`: Информационное сообщение при старте с параметрами (лог уровень, подсеть, порты)

- **run.sh**:
  - Добавлена переменная `SCAN_SUBNET` для передачи подсети из конфига
  - Уровень логирования по умолчанию изменён на `info`

- **config.yaml**:
  - Добавлена опция `scan_subnet` со значением по умолчанию `192.168.1.0/24`
  - Версия обновлена до `1.3.0`

- **index.html**:
  - Функция `loadDefaultSubnet()` для загрузки подсети из конфига при старте
  - Поле ввода подсети автоматически заполняется значением из конфигурации

#### Исправлено
- Проблема с автообнаружением принтеров в разных подсетях
- Проблема с подключением к принтеру при старте
- Некорректная обработка ошибок в DiscoveryService
